# Getting Support

Stuck? Confused by a concept? Can't get an example to run? Here's how to get help — and how to
help us help you faster.

## 🧭 Where to go

| I want to… | Go here |
|------------|---------|
| Ask a question or discuss an idea | [**GitHub Discussions**](https://github.com/bee-ai-labs/bee/discussions) |
| Report a wrong fact, broken link, or bug | [**Open an issue**](https://github.com/bee-ai-labs/bee/issues/new/choose) |
| Request new content on a topic | [**Content request issue**](https://github.com/bee-ai-labs/bee/issues/new/choose) |
| Chat with the community in real time | [**Discord**](https://discord.gg/bee-ai) |
| Report a security problem | [**SECURITY.md**](SECURITY.md) (do **not** open a public issue) |
| Report Code of Conduct concerns | niyitegekatresor@gmail.com |

> [!IMPORTANT]
> **Discussions vs. Issues.** Use **Discussions** for open-ended questions ("how would I…?",
> "which approach is better?"). Use **Issues** for concrete, actionable problems ("this link is
> broken", "this code errors on line 12"). This keeps the issue tracker focused on work.

## 🙋 Asking a great question

You'll get a faster, better answer if you include:

1. **What you're trying to do** — the goal, not just the error.
2. **What you tried** — the command, code, or page you were following.
3. **What happened** — the exact error message (as text, not a screenshot, if possible).
4. **Your environment** — OS, Python version, and which example/page you're on.

A good question is a gift to future learners — it often becomes a new FAQ entry or a fix. 🐝

## 🐝 What we *can't* help with

Bee teaches AI engineering, but we're a community project, not a support desk for third-party
tools. For these, please use the vendor's own channels:

- Billing or account issues with an LLM provider (OpenAI, Anthropic, Google, etc.)
- Bugs in a framework itself (LangChain, LlamaIndex, etc.) — report upstream
- Help with proprietary/internal systems at your company

If a **vendor's docs are the real answer**, we'll point you there — and maybe add a guide so the
next person finds it here first.

## ⏱️ Response times

Bee is maintained by volunteers. We aim to respond to Discussions and issues within a few days.
Being kind and patient goes a long way — and if you can answer someone else's question while you
wait, you're already contributing. 💛
