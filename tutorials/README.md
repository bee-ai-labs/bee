# Tutorials 📝

**End-to-end walkthroughs** — build a complete thing from scratch, step by step, understanding
each part as you go.

> [!NOTE]
> **Tutorials vs. Examples.** A [tutorial](.) *teaches you to build* something incrementally, with
> explanation at each step. An [example](../examples/) is the *finished*, runnable result you can
> clone. Many tutorials end by pointing at a matching example.

## What makes a good tutorial

- **Starts from zero** and builds up in clear steps.
- **Explains the why** at each step, linking to [concepts](../docs/).
- **Ends with something that works** — and ideally a matching [example](../examples/).
- **Checkpoints** so readers can confirm they're on track.

Writing a tutorial is one of the most impactful contributions you can make. See
