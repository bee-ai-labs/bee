# Guides 🧭

**Task-oriented how-tos** — short, focused answers to "how do I do X?" You arrive with a specific
problem and leave with the solution.

> [!NOTE]
> **Guides vs. Tutorials vs. Docs.** A **guide** solves one specific task ("how do I stream
> responses?"). A [tutorial](../tutorials/) walks you through building a whole project. The
> [docs](../docs/) teach the underlying concepts.

## What makes a good guide

- **One task, start to finish.** No detours.
- **Runnable code**, copy-pasteable.
- **Assumes the concept is understood** — link to [docs](../docs/) for the why.
- **Short.** If it's growing into a project, it's a [tutorial](../tutorials/).
